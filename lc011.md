# LC 11 – Container With Most Water

**Difficulty:** Medium
**Link:** https://leetcode.com/problems/container-with-most-water/
**Technique:** Two Pointers

---

## Problem

You are given an integer array `height` of length `n`. There are `n` vertical lines
drawn such that the two endpoints of the `i`-th line are `(i, 0)` and `(i, height[i])`.

Find two lines that together with the x-axis form a container that holds the **most water**.

Return the *maximum amount of water* a container can store.

> The container cannot be slanted.

---

## Examples

```
Input:  height = [1, 8, 6, 2, 5, 4, 8, 3, 7]
Output: 49
Explanation: Lines at index 1 (height 8) and index 8 (height 7).
             Area = min(8, 7) * (8 - 1) = 7 * 7 = 49

Input:  height = [1, 1]
Output: 1
```

---

## Constraints

- `n == height.length`
- `2 <= n <= 10^5`
- `0 <= height[i] <= 10^4`

---

## Approach – Two Pointers

Place one pointer at the left end (`l = 0`) and one at the right end (`r = n-1`).

At each step:
1. Compute area = `min(height[l], height[r]) * (r - l)`
2. Update `best`
3. **Move the pointer at the shorter wall inward.**

**Why move the shorter wall?**
The width shrinks by 1 no matter which pointer we move. The area is bounded by
`min(height[l], height[r])`. Moving the taller wall can only keep that min the same
or make it smaller → area cannot improve. Moving the shorter wall might reveal a
taller line that increases the min and compensates for the lost width.

**Complexity:** O(n) time · O(1) space

---

## Solution

```python
def max_water(height: list) -> int:
    l = 0
    r = len(height) - 1
    best = 0
    while l < r:
        water = min(height[l], height[r]) * (r - l)
        best = max(best, water)
        if height[l] < height[r]:
            l += 1
        else:
            r -= 1
    return best
```
